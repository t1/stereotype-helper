Helps libraries to support stereotypes, so their users can encapsulate the libraries' annotations into their own, allowing to reuse them.

Features Supported

  • {@link Stereotype} (careful: this is our own type!) works just as with CDI: Inherit the library-annotations on the user-annotation to the site-of-use.
  • Annotation parameters from the user-annotation are propagated into your annotations.
  • Type and Field annotations

Features Not Yet Supported

  • Method, etc. annotations
  • Allow adding library-annotations and properties from (xml?) config files (deployment descriptors)
  • Default annotations on classes and packages
